# Sing-box-yg 全面加固设计

## 目标

在不改变现有菜单入口、协议能力和主要使用习惯的前提下，把 `reqingonline/sing-box-yg` 建设为可长期维护、可验证发布、可安全升级和回滚的稳定发行版。设计同时覆盖 VPS 脚本与 Serv00/Hostuno 脚本，并以 Ubuntu 24.04 测试 VPS 的真实安装结果作为发布门槛。

## 成功标准

- 新 VPS 能从本仓库稳定 Release 完成首次安装，安装结束后的 `sb` 仍指向本仓库。
- 配置、脚本或内核更新失败时，旧服务继续可用；不能把失败状态保存为最后可用状态。
- 安装、端口跳跃和卸载只修改本项目拥有的防火墙规则，不清空系统全局链。
- Serv00/Hostuno 重置和卸载只能删除本项目登记的路径，不能递归删除整个用户主目录。
- Token、私钥、节点配置和订阅内容默认仅属主可读，终端和日志不显示完整凭据。
- 公开订阅不使用无保护的明文 HTTP；本地服务默认仅监听回环地址。
- Ubuntu、Debian、Rocky Linux、Alpine 的自动测试执行真实安装路径，而不是只检查包管理器是否存在。
- Ubuntu 24.04 测试 VPS 通过首次安装、升级、失败回滚、重启、防火墙保留和卸载验收。
- `main` 只接收通过 CI 与测试 VPS 验证的稳定提交，并以标签和 GitHub Release 发布。

## 非目标

- 不新增代理协议，不改变客户端产品选择。
- 不重新设计交互菜单或品牌视觉。
- 不在本轮引入 Web 管理后台、用户系统或远程遥测。
- 不对 sing-box、cloudflared 或第三方规则库做功能性分叉。

## 总体架构

保留 `sb.sh`、`serv00.sh` 和 `serv00keep.sh` 作为用户入口，将下载、事务、服务、防火墙、凭据、订阅和清理等高风险操作迁移到职责单一的 Bash 模块。入口脚本负责参数收集、菜单和协议配置生成；模块负责可测试的系统操作。

新增目录：

```text
lib/
  source.sh
  download.sh
  transaction.sh
  firewall.sh
  secrets.sh
  service.sh
  subscription.sh
  cleanup.sh
tests/
  unit/
  integration/
  fixtures/
scripts/
  install.sh
  release-checks.sh
```

模块加载失败必须停止对应操作并保留现有服务。为兼容一行安装，稳定 Release 同时提供包含全部模块的归档；安装器先下载归档、验证清单，再安装入口和模块。仓库源码运行仍支持直接执行 `sb.sh`。

## 模块职责

### `lib/source.sh`

- 定义本仓库所有者、仓库名、稳定通道、版本和 Release API。
- 允许测试通过环境变量覆盖 API 与下载地址，生产默认值固定为 `reqingonline/sing-box-yg`。
- 禁止运行时回源 `yonggekkk/sing-box-yg`；上游只用于维护者同步。
- 区分 `stable`、固定标签和显式测试通道，默认不得自动使用预发布版本。

### `lib/download.sh`

- 所有下载使用 HTTPS、连接超时、总超时、有限重试和失败即停止。
- 可执行文件与脚本必须验证 Release 清单中的 SHA-256；候选脚本还要通过 `bash -n`。
- sing-box 核心继续验证官方 Release API 提供的摘要。
- cloudflared 和本仓库二进制纳入同一校验流程。
- 下载到目标目录内的随机临时文件，校验完成后同文件系统原子替换。
- 使用全局锁防止安装、更新和菜单操作并发写入。

### `lib/transaction.sh`

- 事务对象包含配置文件、sing-box 内核、管理脚本和服务启停状态。
- 变更前保存只读快照，快照目录权限为 `0700`，文件权限为 `0600`。
- 候选内核必须先对候选配置执行 `check`。
- 切换后必须验证服务为 active、预期端口处于监听状态，并完成最小本机连接检查。
- 只有所有验证成功才提交新的最后可用快照。
- 任一步失败都恢复旧配置和旧内核、重新启动旧服务，并以非零状态返回。
- 回滚失败时保留所有诊断材料并输出明确的人工恢复命令，但不得继续覆盖快照。

### `lib/firewall.sh`

- 创建项目专用 `SBYG_PREROUTING` 链，并为规则添加可识别注释。
- 入口链只包含一次跳转；添加、列出和删除操作只针对专用链。
- 同时支持 iptables 与 ip6tables；检测 nftables 后使用兼容路径或明确拒绝不受支持的修改。
- 删除关闭 UFW、停止 firewalld、把策略改为 ACCEPT、清空 filter/mangle/PREROUTING 的功能。
- 默认只显示云安全组与主机防火墙所需的精确 TCP/UDP 端口；自动放行必须由用户显式选择。
- 卸载前后保存并比较非 SBYG 规则，发现变化即视为测试失败。

### `lib/secrets.sh`

- 脚本启动时使用 `umask 077`。
- `/etc/s-box` 中包含凭据的文件统一 `0600`，目录统一 `0700`；公开证书可为 `0644`。
- Argo、Telegram、GitLab 和订阅凭据不得在菜单、日志、进程参数记录或 Git remote URL 中完整显示。
- GitLab 使用权限最小化的令牌；本地保存时单独置于凭据文件，不拼入仓库 remote。
- 提供统一脱敏函数，只显示凭据首尾少量字符。
- CI 扫描仓库与测试输出，阻止真实 Token、私钥和订阅密钥进入提交或构件。

### `lib/service.sh`

- 首次启动、重启和升级统一走事务验证，不允许直接 `start` 后再盲目 `restart`。
- systemd 使用 `network-online.target`，按实际功能收敛 capabilities，并启用 `NoNewPrivileges`、`PrivateTmp`、合理的文件系统保护。
- OpenRC 提供同等的启动结果检查。
- 删除每日无条件重启 cron，改为健康检查 timer；健康检查只在配置合法且服务异常时恢复。
- 健康检查记录版本、服务状态、配置摘要和监听状态，不记录节点凭据。

### `lib/subscription.sh`

- BusyBox HTTP 服务默认绑定 `127.0.0.1`，不能直接监听公网。
- 公网订阅必须通过 HTTPS 反向代理或受控隧道提供，并明确把订阅 URL 视为凭据。
- 订阅路径使用独立随机令牌，不复用节点 UUID。
- 订阅目录禁止目录列表，文件权限遵循最小授权。
- 变更订阅端口或令牌时采用原子切换，旧服务停止后才清理旧目录。

### `lib/cleanup.sh`

- 安装时写入项目资产清单和标记文件，记录本项目创建的文件、目录、服务和防火墙链。
- 卸载只处理清单中且仍位于允许根目录内的路径。
- Serv00/Hostuno 禁止对 `$HOME`、`~` 或未解析变量执行递归删除。
- 删除前解析绝对路径、拒绝越界目标、显示清单并要求显式确认。
- 不修改非项目文件权限，不杀死同一用户的所有进程，只终止记录 PID 或可验证命令行的项目进程。

## 配置兼容

- 版本解析使用语义版本比较，不再以“1.10 或其他”二分。
- 保留 1.10.7 兼容配置，同时为当前稳定系列和 1.14+ 配置分别生成经过校验的模板。
- 1.14+ 配置显式定义 `http_clients` 与 `route.default_http_client`，不使用已废弃的 `download_detour`。
- 每个生成模板都必须由对应版本的 sing-box 二进制执行 `check`。
- 上游提交先合入同步分支，生成差异报告并通过全部测试后才能进入稳定分支。

## 输入验证

- 端口必须是 `1..65535` 的十进制整数，端口范围必须满足起点小于终点。
- 同一协议与跨协议端口冲突在写配置前检测。
- 域名、分支名、项目名、路径和 UUID 分别使用白名单验证，禁止把未经验证的输入插入 shell、sed、expect 或 `bash -c`。
- 所有 `read` 使用 `-r`，变量引用默认加双引号。
- 大型旧脚本不直接启用全局 `set -e`；新模块使用严格模式并显式传播返回值，入口逐步收敛错误处理。

## 安装、更新与回滚流程

### 首次安装

1. 下载固定标签的安装器和校验清单。
2. 校验安装器，检测操作系统、架构、磁盘和依赖。
3. 只安装缺失依赖，不执行整个系统的无条件升级。
4. 生成候选配置和证书，设置安全权限。
5. 下载并验证候选内核。
6. 候选内核校验候选配置。
7. 安装服务并启动，验证状态与监听端口。
8. 记录资产清单和最后可用快照。
9. 输出需要在云安全组和系统防火墙开放的精确端口。

### 更新

1. 获取全局锁并读取当前版本。
2. 下载目标 Release 元数据、归档和校验清单。
3. 保存配置、内核、脚本和服务状态快照。
4. 验证候选组合后原子切换。
5. 完成服务、监听和基础连接验证。
6. 成功后更新版本标记；失败时自动回滚。

### 卸载

1. 读取资产清单并解析所有目标。
2. 显示服务、文件、目录和专用防火墙链清单。
3. 停止项目服务并删除专用规则。
4. 删除清单内资产，保留未知文件并报告。
5. 验证 SSH、非项目防火墙规则和其他服务仍正常。

## 测试设计

### 单元测试

- 下载超时、空文件、摘要错误、语法错误和原子替换。
- 端口、范围、UUID、域名、分支与路径验证。
- systemd/OpenRC 启动失败后的返回值与回滚顺序。
- 配置合法但服务启动失败时不能提交快照。
- 内核升级失败恢复旧二进制与旧配置。
- 专用防火墙链创建、幂等更新和精确删除。
- 凭据文件权限、脱敏输出和仓库来源固定。
- 清理路径越界、符号链接和空变量拒绝。

### 容器集成测试

- Ubuntu 24.04、Debian 12、Rocky Linux 9、Alpine 3.20。
- 每个镜像执行依赖安装、候选配置生成、模块安装、服务定义生成和卸载检查。
- systemd 相关测试使用适配容器；OpenRC 在 Alpine 容器验证。
- 容器退出前确认没有修改非项目防火墙规则和非项目文件。

### 测试 VPS 验收

- 记录初始 SSH、UFW、iptables/nftables、监听端口和系统服务基线。
- 从稳定候选 Release 完成真实首次安装。
- 验证 IPv4/IPv6、各协议 TCP/UDP 监听和 sing-box 配置。
- 注入坏配置，确认服务不中断且配置恢复。
- 注入不可启动但语法合法的配置，确认运行期失败回滚。
- 注入坏内核和错误摘要，确认旧内核继续运行。
- 创建非项目防火墙规则后执行端口跳跃与卸载，确认规则保留。
- 验证敏感文件权限和日志脱敏。
- 完成卸载后确认 SSH 会话、UFW 和系统基础服务正常。

## CI 与供应链

- GitHub Actions 默认 `permissions: contents: read`。
- 第三方 Action 固定到审核过的提交 SHA，并定期更新。
- 执行 `bash -n`、ShellCheck、测试矩阵、敏感信息扫描和差异空白检查。
- Release 工作流生成源码归档、校验清单和变更说明；发布前必须有测试 VPS 验收记录。
- 移除仓库中的过时客户端 ZIP；大型自有二进制移到 Release Assets，并附来源、版本、构建说明和摘要。
- `main` 设置分支保护，要求 CI 通过后才能合并。

## 迁移策略

- 现有安装检测到旧 `/usr/bin/sb` 时，先备份后安装模块化管理脚本。
- 读取并保留现有节点配置、端口、证书和订阅设置，不强制重装。
- 首次迁移执行配置校验和服务检查，成功后才建立新资产清单与快照。
- 无法安全识别的旧防火墙规则只报告，不自动删除。
- README 同时提供稳定安装、固定版本安装、从上游版迁移、升级和人工回滚说明。

## 发布与维护

开发流程为：功能分支 → CI → 测试 VPS → 候选标签 → 稳定 Release → `main`。每次发布记录支持系统、sing-box 版本、迁移说明、校验摘要和已知限制。

上游更新通过独立同步分支引入。维护者先比较配置模板、下载地址和菜单行为，再执行容器与 VPS 回归；不能使用 GitHub 的一键同步覆盖本仓库加固代码。

## 可观测性

- 提供 `sb doctor`，汇总系统版本、脚本版本、内核版本、配置检查、服务状态、监听端口、防火墙链和证书有效期。
- 默认输出不包含 UUID、Token、私钥和完整订阅 URL。
- 健康检查失败时保留最近诊断日志，并给出回滚是否成功的明确结论。
- Telegram 告警只发送状态与错误摘要，不发送节点配置。

## 验收结论

只有在全部 CI 通过、测试 VPS 完成首次安装与所有故障注入、SSH 和非项目防火墙规则保持正常、README 安装命令从稳定 Release 验证成功后，全面加固工作才可标记完成。
